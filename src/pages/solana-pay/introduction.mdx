# Introduction

<Callout emoji="⚠️">
  This series is currently incomplete. The first few chapters are available, but the rest are still being written. We expected to be finished by January 2023.
</Callout>


This 10-part series on advanced Solana Pay development you will teach you how to create a Solana token faucet that distributes SPL tokens via a QR code. 
While the faucet is cool, the real goal is to teach the Solana community some of the techniques that are used to create production-ready applications powered by Solana Pay.

This tutorial is geared toward developers who are already familiar with [Solana](https://solana.com) and have a basic understanding of [Solana Pay](https://www.solanapay.com). If you're new to either, checkout these awesome resources:

This series covers the following topics:
<ul>
  <li>
    1. [Introduction](/solana-pay/introduction)
  </li>
  <li>
    2. [Getting Started](/solana-pay/getting-started) - Clone the starter project and set up the environment.
  </li>
  <li>
    3. [Durable status tracking](/solana-pay/durable-tracking) - Add durable transaction-status tracking using a Redis task-queue.
  </li>
  <li>
    4. [Real-time status updates](/solana-pay/real-time-updates) - Use websockets to send real-time status updates to the client.
  </li>
  <li>
    5. [Creating Tokens](/solana-pay/creating-tokens) - Create and issue SFTs and NFTs during a transaction request.
  </li>
  <li>
    6. [Token Gating](/solana-pay/token-gating) - Using on-chain data to approve or deny transaction requests.
  </li>
  <li>
    7. [Off-chain Gating](/solana-pay/off-chain-gating) - Using off-chain data to approve or deny transaction requests.
  </li>
  <li>
    8. [Error Handling](/solana-pay/error-handling) - Using the new Solana Pay error handling specification to pass useful errors to a wallet.
  </li>
  <li>
    9. [Message Signing](/solana-pay/message-siging) - Using the new Solana Pay message signing specification to sign arbitrary messages.
  </li>
  <li>
    10. [QR Code Authorization](/solana-pay/authorization) - Authenticating users via Solana Pay.
  </li>
</ul>
